基于我们刚创建的 Astro 博客项目，现在需要配置 Content Collections 和示例文章。

1. 请在项目根目录创建 content/config.ts，定义一个名为 'blog' 的集合，其 schema 包含：
   - title (string, 必填)
   - description (string, 必填)
   - pubDate (date, 必填)
   - status (enum: 'seed' | 'growing' | 'evergreen', 必填，默认 'growing')
   - tags (array of strings, 可选)

2. 创建以下 3 篇示例 Markdown 文件在 content/blog/ 下：

文章1: `hello-garden.md`
Frontmatter: title: "欢迎来到我的数字花园", description: "这是第一篇文章", pubDate: 2023-10-01, status: "evergreen", tags: ["思考", "元"]
正文: 随便写一段欢迎语，并包含一个指向 [[第二篇文章的标题]] 的双向链接占位。

文章2: `design-system.md`
Frontmatter: title: "设计系统构建", description: "关于设计的思考", pubDate: 2023-10-10, status: "growing", tags: ["设计"]
正文: 随便写一段关于设计的文字。

文章3: `ai-coding.md`
Frontmatter: title: "AI辅助编程", description: "用AI写代码的体验", pubDate: 2023-10-20, status: "seed", tags: ["技术", "AI"]
正文: 随便写一段关于AI编程的文字。

请给出 content/config.ts 的代码，以及这三篇 markdown 文件的完整内容。
