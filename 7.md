现在为博客添加核心的数字花园功能：双向链接与悬浮预览。

当前痛点：Markdown 中的 [[文章标题]] 目前只会被当作纯文本渲染。

要求：
1. 创建 Astro 集成/Remark 插件（在 astro.config.mjs 中引入），在构建时解析 Markdown 中的 [[文章标题]] 语法，将其替换为：<a href="/blog/slug" class="wikilink" data-slug="slug">文章标题</a>。
2. 创建 React 组件 src/components/HoverPreview.tsx：
   - 监听页面上所有带有 .wikilink 类的链接的 mouseenter 事件。
   - 获取 data-slug，请求对应文章的摘要数据（可以在构建时生成一个包含所有文章 title/description/slug 的 JSON 文件 /api/posts.json，或直接写进 data 属性）。
   - 在鼠标旁边渲染一个毛玻璃风格的悬浮卡片，显示文章标题和描述。
   - 鼠标离开链接时延迟消失。
3. 在 src/layouts/BaseLayout.astro 中引入此 React 组件（确保 client:load）。
4. 在 src/pages/blog/[slug].astro 底部添加“反向链接”模块：
   - 在构建时遍历所有文章，找出正文中包含当前文章标题的 [[链接]] 的其他文章。
   - 如果有，显示“链接到这篇文章”的列表。

请给出完整的代码修改，包括 Remark 插件的写法、HoverPreview 组件、以及修改后的布局和详情页。
